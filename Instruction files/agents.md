# AGENTS.md

## Purpose
This project contains Bible/Torah high school lessons for family or small-group study. All lessons must follow the curriculum outline file in this folder and match the structure, tone, and formatting style of the existing lesson examples.

## Source Priority
When generating or revising lessons, use sources in this order:

1. The curriculum scope/outline file in this folder
2. The existing completed lesson files as structure and style examples
3. This AGENTS.md file
4. BELIEFS.md for worldview and doctrinal boundaries

If any source seems to conflict, follow the curriculum outline for sequence, and BELIEFS.md for theological perspective.

## Core Working Rules
- Do not use mainstream Christian framing, assumptions, or church-centered language unless a source file explicitly requires quoting or comparing it.
- Do not write from a replacement theology perspective.
- Do not describe Torah as abolished, replaced, fulfilled in the sense of ended, or made irrelevant.
- Do not frame obedience and grace as opposites.
- Do not insert denominational labels into the lessons unless the lesson explicitly discusses them for contrast.
- Use scripture-centered language.
- Keep wording clear, readable, and natural for teens.
- Stay consistent with the worldview in BELIEFS.md.
- When uncertain, flag the issue instead of inventing theology or drifting into mainstream Christian language.

## Lesson Goals
Lessons should help students:
- understand scripture in context
- grow in love for Yehovah
- see the unity of Torah, Prophets, Writings, and the New Testament writings
- understand the faithfulness, character, commandments, and promises of Elohim
- connect knowledge with faithful living

## Tone and Audience
- Audience: high school level students
- Tone: warm, respectful, scripture-focused, thoughtful, clear
- Avoid academic heaviness unless the lesson calls for it
- Avoid preachy or confrontational wording
- Lessons are for instruction, reflection, and growth, not for arguing against outsiders

## Style Requirements
Use the existing lesson files as the main model for:
- file naming
- layout
- section order
- pacing
- reading level
- formatting
- quiz/question style
- reflection style

Unless the outline says otherwise, keep consistency with the current lesson examples.

## Writing Standards
- Use “Yehovah” where appropriate instead of “the Lord” in original instructional writing
- Use “Yeshua” instead of “Jesus” in original instructional writing
- Use “Torah” and “Tanakh” accurately
- When referencing the New Testament, present it as fully consistent with the earlier scriptures
- Emphasize continuity across scripture
- Define important terms simply
- Keep explanations rooted in scripture rather than tradition

## Scripture Use
- Base claims on scripture
- Do not force citations into every sentence if the current lesson style does not do that, but keep the content scripture-grounded
- When interpreting a passage, prefer cross-scriptural consistency
- Avoid speculative claims unless clearly labeled as possibilities
- Avoid presenting disputed interpretations as certain unless the source files already establish that approach

## What Not to Do
- Do not write as though Christianity or Judaism is the governing framework for the lesson
- Do not argue that the commandments were done away with
- Do not present Paul, Hebrews, or the New Covenant as overturning Torah
- Do not use anti-law language such as “we are no longer under the law so obedience is unnecessary”
- Do not import catechism-style or sermon-style church wording
- Do not overuse phrases like “Old Testament vs New Testament” in a way that implies contradiction
- Do not rewrite existing lessons unless asked

## Workflow for New Lessons
When asked to create lessons:
1. Identify the next lesson or lessons from the curriculum outline
2. Briefly state which lesson numbers/titles come next
3. Follow the established structure from the existing lesson files
4. Match naming conventions already used in the folder
5. Create only the requested number of lessons
6. Stop and note anything unclear rather than guessing

## Revision Rules
When editing existing lessons:
- preserve the project worldview
- preserve formatting unless asked to improve it
- improve clarity, consistency, flow, grammar, and fidelity to the project standards
- do not “normalize” wording into mainstream Christian language

## File Creation Rules
- Follow the existing naming pattern exactly or as closely as possible
- Keep lesson titles descriptive and consistent
- Do not create extra files unless asked
- If creating support files, use simple names such as `lesson-template.md`, `lesson-3-...html`, etc., based on the existing project pattern

## If Uncertain
If there is ambiguity in theology, lesson scope, wording, or format:
- do not guess
- do not default to mainstream Christian interpretations
- note the uncertainty clearly and proceed only where the outline and examples support it
